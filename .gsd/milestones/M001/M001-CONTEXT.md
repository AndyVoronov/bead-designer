# M001: 3D-конструктор игрушек из бусин

**Gathered:** 2026-03-29
**Status:** Ready for planning

## Project Description

Мобильное веб-приложение для конструирования детских игрушек из бусин с реалистичным 3D-редактором и физикой. Клиент на телефоне собирает изделие (держатель для соски и далее другие типы), добавляя бусины разных форм, размеров и материалов на нить. 3D-физика позволяет «потрогать» изделие — перетаскивать бусины, трясти, крутить. Один тап «Заказать» → заказ в БД → Telegram с менеджером. Каждое изделие = уникальная ссылка. Админка для управления контентом.

## Why This Milestone

Это первый и основной milestone — от пустого проекта до работающего продукта, доступного клиентам на мобильных. Включает все ключевые возможности: 3D-редактор с физикой, каталог, шаблоны, заказы, админку, деплой.

## User-Visible Outcome

### When this milestone is complete, the user can:

- Открыть сайт на телефоне и увидеть каталог шаблонов изделий
- Выбрать шаблон или начать с нуля — собрать держатель для соски из бусин
- Перетаскивать бусины пальцем в 3D-сцене с реалистичной физикой (гравитация, провисание нити, столкновения)
- Нажать «Заказать» → открыть Telegram с менеджером с готовым сообщением
- Поделиться ссылкой на своё изделие — другой человек увидит точную копию
- (Админ) Управлять шаблонами, просматривать заказы, подтверждать дизайны

## Risks and Unknowns

- **Мобильная производительность 3D + физика** — WebGL + WASM на мобильных браузерах может тормозить с множеством бусин и пост-обработкой. R3F PerformanceMonitor + dynamic DPR смягчают, но не гарантируют.
- **Стабильность rope physics** — цепочка из 20-40 бусин на spring/rope joints может быть нестабильной при быстрых перетаскиваниях. Rapier обычно стабилен, но edge cases возможны.
- **Реалистичные материалы из PNG** — deriving PBR-параметров из фотографий бусин — нетривиальная задача. Возможно потребуется ручная настройка normal/roughness maps для каждого типа материала.
- **Размер бандла** — Three.js + Rapier WASM = большой JS bundle. Tree-shaking и code splitting критичны для мобильных.

## Technical Constraints

- Target: мобильные браузеры (Chrome, Safari на iOS). Минимально — средние устройства 2022+.
- Физика: WASM (Rapier). Не переключаемся на CPU-only — качество важнее поддержки старых устройств.
- Текстуры: PNG-изображения от пользователя. Нужно уметь загружать и назначать как PBR-текстуры.
- База данных: PostgreSQL (через Prisma ORM).
- Хостинг: VPS на reg.ru (Node.js).
- Telegram: deep link https://t.me/VoronovAndrey?text=... — без бота, просто предзаполненный чат.

## Integration Points

- **PostgreSQL** — хранение бусин, шаблонов, заказов. Prisma ORM, Next.js API routes.
- **Telegram** — ссылка на чат с менеджером. Без API-интеграции, просто URL redirect.
- **Файловая система** — PNG-текстуры бусин. Хранение в /public/beads/ или через CDN.

- **Детали админки** — нужен ли импорт бусин через UI или хватит seed-файла? Решается при проектировании S06.
